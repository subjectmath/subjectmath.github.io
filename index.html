Web Worker

Posted on 2018-11-09

一、概述

单线程的js，当涉及到复杂的js运算时可能会短暂性“冻结”，不能执行其他操作。

而Web Worker的作用就是为JavaScript创造多线程环境，可以将耗时任务拆解出去，降低主线程的压力，避免主线程无响应。

创建Web Workers
收发消息

Web Workers用来执行异步脚本，只要掌握了它与主线程通信的方式。

  1. 向worker发消息，数据以异步传递给worker，触发worker中的message事件

  2. Worker处理接受消息

  3. 主线程处理返回的数据

  4. 如果worker不能完全给定任务则会触发error事件

写个node服务

node server.js开启本地服务器，然后就可以使用worker了，可以发现在计算的时候页面没有卡住啧啧啧

缺点

  1. 分配给Worker线程运行的脚本文件必须与主线程的脚本文件同源
  2. 不能操作Dom
  3. 不能读取本地文件，只能来源于网络
  4. Worker线程不能执行alert()和confirm()方法，但可以使用xhr发送ajax请求

    补漏

  5. 在index.js中可以使用 worker.terminate() 来终止一个 worker 的执行
  6. 在worker.js中可以使用 self.close() 来终止一个 worker 的执行。

Promise 承诺

异步：操作之间没有关系，同时进行多个操作
同步：同时只能做一件事

异步：代码复杂
同步：代码简单

Promise—-消除异步操作

  • 用同步一样的方式，来书写异步代码

Promise到底怎么用

Promise.prototype.then();

第一个参数是resolved状态的回调函数，第二个参数（可选）是rejected状态的回调函数。

Promise.prototype.catch();

等于.then(null,rejection)，用于指定发生错误的回调函数

Promise.prototype.finally();

不管状态如何，都会执行

4.关于Dom模板的解析

由于有些Dom元素的父子直接是有联系的，比如父ul或ol和子li，父table和子tr或td或th,父selct和子option。

组件的属性和事件

分析：

  • foo是组件内部定义的一个prop属性，baz是父组件的一个data属性
  • event-a是子组件的一个事件，doSomething是父组件的一个方法

过程：

  • 父组件把baz数据通过prop传递给子组件的foo
  • 子组件内部得到foo的值，就可以进行相应的操作
  • 当子组件内部发生变化希望父组件能知道，就利用代码触发event-a事件，把一些数据发送出去
  • 父组件把该事件处理器绑定为doSomething方法，子组件发送的数据，就作为doThis方法的参数被传进来
  • 然后父组件根据这些数据进行操作

属性Props

1.Vue组件通过props属性声明一个自己的属性，然后父组件就可以往里面传数据

2.动态绑定特性值(v-bind)

  • 当父组件传递的属性是引用类型时，在子组件中修改相应的属性会导致父组件相应属性的改变。
  • 当父组件传递值为基本类型时，在子组件中改变这个属性会报错，要在父组件加上.sync修饰符。

然后就可以在子组件中改变相应属性了。

探究v-model

v-model可以对表单控件实现数据的双向绑定，它的原理就是利用了绑定属性和事件的实现。比如input控件，不使用v-model，可以这样实现数据的双向绑定

  • 把input的value特性绑定到Vue实例的属性text上，text改变，input中的内容也会改变
  • 然后把表单的input事件处理函数设置为Vue实例的一个方法，这个方法会根据输入参数改变Vue中的text的值。
  • 相应的，在input中输入内容时，触发了input事件，把event.target.value传给这个方法，最后就实现了改变绑定数据的效果。
