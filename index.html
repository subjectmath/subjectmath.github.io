组件的属性和事件

分析：

  • foo是组件内部定义的一个prop属性，baz是父组件的一个data属性
  • event-a是子组件的一个事件，doSomething是父组件的一个方法

过程：

  • 父组件把baz数据通过prop传递给子组件的foo
  • 子组件内部得到foo的值，就可以进行相应的操作
  • 当子组件内部发生变化希望父组件能知道，就利用代码触发event-a事件，把一些数据发送出去
  • 父组件把该事件处理器绑定为doSomething方法，子组件发送的数据，就作为doThis方法的参数被传进来
  • 然后父组件根据这些数据进行操作

属性Props

1.Vue组件通过props属性声明一个自己的属性，然后父组件就可以往里面传数据

2.动态绑定特性值(v-bind)

  • 当父组件传递的属性是引用类型时，在子组件中修改相应的属性会导致父组件相应属性的改变。

然后就可以在子组件中改变相应属性了。

探究v-model

v-model可以对表单控件实现数据的双向绑定，它的原理就是利用了绑定属性和事件的实现。比如input控件，不使用v-model，可以这样实现数据的双向绑定

  • 把input的value特性绑定到Vue实例的属性text上，text改变，input中的内容也会改变
  • 然后把表单的input事件处理函数设置为Vue实例的一个方法，这个方法会根据输入参数改变Vue中的text的值。
  • 相应的，在input中输入内容时，触发了input事件，把event.target.value传给这个方法，最后就实现了改变绑定数据的效果。

变量的解构赋值(声明和赋值不能分开)

1.数组的解构赋值（位置上的对应很重要）

  1. 只要对应上就可以，数据的类型可以多样化，对象也可以，按位赋值，没有赋到值的变量为undefined

  2. 只要某种数据结构具有 Iterator 接口，都可以采用数组形式的解构赋值。
    //Generator函数（后面再补）
    //yield??????（后面再补）

  3. 指定默认值(右边没有值与前面变量对应的时候,或者是undefined的时候)

  4. 如果默认值是表达式，则这个表达式是==惰性求值==的，只有在用到的时候，才会求值

2.对象的解构赋值（变量名字对应很重要）

  1. 对象的解构与数组不同的是，数组元素是依次序排列的，变量取值与位置有关，而对象的属性没有次序，变量必须与属性同名，才能取到正确值

  2. 其实上面的解构是，变量名和属性名一致，也可能不一致

  3. 对象（加数组）的嵌套结构,如果解构模式是嵌套的对象，而且子对象所在的父属性不存在，那么将会(读取undefined的属性)报错。

  4. 对象的解构也可以指定默认值(默认值生效的条件是，对象的属性值===undefined。)

3.字符串的解构赋值

字符串被转换成一个类似数组的对象

4.数值和布尔值的解构赋值

先把数值和布尔值变成包装对象，再进行对象的解构赋值

undefined和null不能化为对象，对它们解构赋值会报错

5.函数参数的解构赋值

函数参数的解构使用默认值（对x和y设置默认值）

函数的参数对象使用默认值（对数组参数[x, y]设置默认值）

6.圆括号

  1. 只要有可能导致解构的歧义，就不得使用圆括号。
  2. 不能适应圆括号的情况：
    （1）变量声明语句 ：let [(a)] = [1];
    （2）函数参数： function f([(z)]) {return z;} //函数参数也属于变量声明
    （3）赋值语句的模式： ({ p : a}) = {p:42} //p即为模式。
  3. 可以使用圆括号的情况：赋值语句的非模式部分 [(b)] = [3]; 并没有声明，只是赋值。
