canvas

1.canvas中的save()与restore()

1.save()方法所说的绘图状态：当前坐标系统（绘图原点是否更改），旋转角度（画布的旋转角度），填充风格，线条风格，阴影风格。

2.restore()是指回到对应上一次设置save()的地方前所保留的绘图状态。

3.通过save()和restore()使得我们每次操作画布中的部分元素时，都可以在操作完该元素后重新回到初始的绘图状态从而不会因前者绘图状态的变化而使后来的元素受到影响。

2.清除画布，使得canvas图像持续变化

3.canvas小demo

canvas时钟实现原理是画出表上的刻度与数字作为静态背景，分别做绘制时针分针和秒针的函数，new日期变量，将时间的小时分钟和秒钟分别传进对应的函数参数内。在分别设定好三只指针的长度后，通过时刻算出对应指针的旋转角度，利用sin与cos还有半径便可以算出对应时刻三只指针的坐标。通过lineTo()即可绘制成功。通过定时更新画布并且动态改变指针绘制情况，实现时钟的基本功能。

let 和 const

let

1.let所声明的变量只在let命令所在的代码块{}内有效

2.for循环适合用let声明变量（不会让计数变量泄露为全局变量），for循环变量部分是一个父作用域，而循环体是一个单独的子作用域

3.let不存在变量提升，声明变量之前使用会报错，而不像var是把变量设定为undefined；let变量不能重复声明，不能在函数内部重新声明参数

4.暂时性死区(temporal dead zone)：let前变量不可用（报错），typeof操作也不行（报错）不同于var的undefined

5.在全局作用域下let声明的不是window的属性了,const声明也一样

块级作用域

1.ES6引入块级作用域，块级作用域内声明函数类似let，在块级作用域之外不可引用，不会提升。（在浏览器的 ES6 环境中，块级作用域内声明的函数，行为类似于var声明的变量。）

const

1.const实际上保存的是指针，要求指针所指的东西不能改变，但如果是对象，却可以在上面改属性，但改变对象的话依然会报错

1.容器默认主轴（main axis）和垂直的交叉轴（cross axis）

2.最常用的属性

1.justify-content属性设定了项目在主轴（水平）上的对齐方式

2.了解bootstrap的实现原理

实现的基本原理是利用媒体查询，根据不同设备上的屏幕宽度的像素的不同使用不同的已定义好的类名进行不同的布局状态

行（row）”必须包含在.container（固定宽度）或 .container-fluid（100% 宽度）的div中,即上一级div得设置这两个类名中的一个

3.事件冒泡，事件捕获和事件委托

1.事件冒泡

事件从最小的元素的事件对应的操作开始执行，后由于冒泡，其父元素或祖先元素对应的操作再继续执行，即从里面往外面触发事件。

2.事件捕获

事件从最外面的元素的事件对应的操作开始执行，后再逐渐深入，其子元素或孙元素对应的操作再继续执行，即从外面往里面触发事件

3.事件委托

主要目的是为了减少浏览器对于dom的操作从而提高性能，以ul下多个li做例子，实现的原理是li的点击事件冒泡到ul的点击事件上，再对ul的点击事件进行判断确定是哪个li的点击事件

搭建Hexo与了解Git基本操作

Posted on 2018-07-12

1.搭建hexo

1.安装Git Bash 与 node.js

$ git version 有版本号则为安装成功

$ node -v 有版本号则为安装成功

2.用来保存博客的文档

本地新建文档创建，用npm安装hexo

$ npm i -g hexo

node_modules：是依赖包

public：存放的是生成的页面

scaffolds：命令生成文章等的模板

source：用命令创建的各种文章

themes：主题

_config.yml：整个博客的配置

db.json：source解析所得到的

package.json：项目所需模块项目的配置信息

3.将本地的hexo搭载到自己的github上

1.在自己的github上头像左边加号里面new一个repository（仓库）并设置为yourname.github.io

2.在本地gitbash中设置全局的账户信息

$ git config –global user.name “subj…”（自己设置）

$ git config –global user.email “15..@qq.com“（自己设置）

3.在自己本地生成ssh并且放到github上去

$ ssh-keygen -t rsa -C “15..@qq.com@example.com” 生成ssh

$ cd ~/.ssh

$ cat id_rsa.pub 然后将下面出门的整个东西放到github上setting中自己创建的SSH keys 里面

4.在gitbash中验证是否添加成功

$ ssh -T git@github.com

5.编辑器打开hexo文档下的_config.yml配置(对应你的github名字填写)

配置完成后，在gitbash中启动

可以在本地的浏览器输入localhost:4000,就可以看到hexo/source/_posts下的文档了（目前是用markdown的语法写的blog）

6.上传到github

$ npm install hexo-deployer-git –save 将你写的文章发布到github上开元

执行命令

然后在浏览器输入http://yourgithubname.github.io（对应github名自己改）就可以通过github看到自己的博客了

2.用git实现合作开发的文件改动

克隆git上的文件

$ git clone (在github复制下来)

打开他的文件包

$ cd first

检查是否接上分支

$ git remote -v

把主分支上的文件拉下来

$ git pull origin master

创建并进入你的分支

$ git checkout -b jiajun

查看你的支路状态

$ git branch

查看你所在支路的状态

$ git status

将修改的文件添加到自己所在分支

$ git add jiajun.txt

将修改的文件推送到自己所在分支

$ git commit -m “jiajun”

将自己分支上的文件拉下来

$ git push origin jiajun

进入主线路

$ git checkout master

把主线路的文件拉下来

$ git pull origin master

再把自己线路上的文件推送到主路线上

$ git merge jiajun
